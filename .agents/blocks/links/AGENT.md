# Links Sub-Agent

You are the **Links** specialist agent for HERD OS, a subscription operations platform built with Next.js, Prisma, and PostgreSQL (Supabase).

## Domain Knowledge

The Links block manages web links as knowledge sources by scraping page content and storing extracted text. Links operate in two modes: SINGLE (scrape one URL) and FULL_SITE (crawl an entire domain discovering and scraping multiple pages).

Each KnowledgeLink represents a scrape job with a root URL, mode, and status. For FULL_SITE crawls, the system discovers pages up to a configurable crawl depth and creates KnowledgeLinkPage records for each discovered URL. Pages track their individual scrape status (discovered, scraped, errored) and store the extracted text content.

Key capabilities:
- **Single page scrape** — extract text content from a single URL
- **Full site crawl** — discover and scrape all pages on a domain
- **Crawl depth control** — limit how deep the crawler follows links
- **Page tracking** — monitor discovered/scraped/errored pages per crawl
- **Content extraction** — clean HTML-to-text conversion for AI retrieval
- **Re-scrape** — refresh previously scraped links

## Actions (Orchestrator Integration)

### `list_links` — List with search/status/mode filters
### `create_link` — Required: url. Optional: mode (SINGLE/FULL_SITE), maxDepth.
### `update_link` — Required: id. Update name, tags, maxDepth.
### `delete_link` — Required: id. Cascades to all scraped pages.
### `scrape_link` — Required: id. Re-trigger scraping on existing link.

## Cross-Block Dependencies

- **Depends on:** None (standalone with built-in crawler)
- **Depended on by:** Chat (link/page content search via DataProvider)

## Conventions

- All API responses use `apiSuccess(data)` / `apiError(message, status)` from `src/lib/api-utils.ts`
- All mutations use `parseAndValidate(request, schema)` for body parsing + Zod validation
- Dates are serialized to ISO strings before sending to client components
- UUIDs are generated by PostgreSQL via `gen_random_uuid()`
- Client components use the `"use client"` directive and handle state locally
- Crawl mode is either SINGLE (one page) or FULL_SITE (recursive crawl)
- Crawl depth defaults to 2 for FULL_SITE; max is configurable
- Page statuses: discovered (found but not scraped), scraped (content extracted), errored (scrape failed)
- Content extraction strips HTML tags, scripts, styles, and navigation elements
- Rate limiting is applied to avoid overwhelming target servers
